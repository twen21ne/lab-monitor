# LAB Monitor

Простая система мониторинга LABUSDT (Binance Futures) на бесплатном GitHub Actions.
Следит за funding rate, изменением open interest и резкими движениями цены (приближённо — как прокси ликвидаций),
шлёт алерты в Telegram.

## Важная честная оговорка

Точный публичный REST-эндпоинт по объёму ликвидаций Binance закрыл для программного опроса по расписанию
(доступен только как непрерывный WebSocket-поток, который GitHub Actions не может держать открытым между запусками).
Поэтому метрика №3 в этой версии — это **приближение**: резкое движение цены (>5% за 15 минут) вместе с падением
open interest (>5% за то же окно) в реальности почти всегда означает каскад ликвидаций, но это не точная цифра в $.
Если позже захочешь точные суммы ликвидаций — понадобится either платный CoinGlass API, либо отдельный
постоянно работающий сервер с открытым WebSocket (не GitHub Actions).

Холдер-концентрацию (топ-10 кошельков) в эту автоматическую версию сознательно не включил — BscScan сейчас
меняет API (старые бесплатные эндпоинты депрекейтятся, новый Etherscan API V2 не бесплатен для BNB Chain).
Эта метрика и так медленная (смотреть раз в сутки), так что предлагаю смотреть её вручную раз в несколько дней:
https://bscscan.com/token/0x7ec43cf65f1663f820427c62a5780b8f2e25593a#balances

## Шаг 1. Создать Telegram-бота

1. В Telegram напиши **@BotFather** → команда `/newbot` → придумай имя → получишь `TELEGRAM_BOT_TOKEN`
   (выглядит как `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`)
2. Напиши своему новому боту любое сообщение (просто "привет"), чтобы у бота появился с тобой чат
3. Узнай свой `TELEGRAM_CHAT_ID`: открой в браузере
   `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates` (подставь токен)
   и найди в ответе `"chat":{"id": ЧИСЛО, ...}` — это и есть chat_id

## Шаг 2. Создать репозиторий на GitHub

1. Зайди на github.com → New repository → назови, например, `lab-monitor` → **можно private, можно public**
   (для приватного репозитория GitHub Actions даёт 2000 бесплатных минут в месяц — этого более чем достаточно;
   для публичного репозитория Actions вообще бесплатны без лимита)
2. Залей туда 4 файла из этого проекта, сохранив структуру папок:
   ```
   lab-monitor/
   ├── monitor.py
   ├── state.json
   ├── README.md
   └── .github/workflows/monitor.yml
   ```
   Проще всего — через веб-интерфейс GitHub: "Add file" → "Upload files", перетащить всё разом
   (GitHub сам создаст папку .github/workflows, если загрузить с сохранением пути)

## Шаг 3. Добавить секреты (токены) в репозиторий

1. В репозитории: Settings → Secrets and variables → Actions → New repository secret
2. Добавь два секрета:
   - `TELEGRAM_BOT_TOKEN` = токен из шага 1
   - `TELEGRAM_CHAT_ID` = chat_id из шага 1

Секреты не видны никому, даже тебе после сохранения — только сам workflow может их использовать.

## Шаг 4. Проверить, что всё работает

1. Перейди во вкладку **Actions** репозитория
2. Слева выбери workflow "LAB Monitor"
3. Справа нажми **Run workflow** → **Run workflow** (запуск вручную, не дожидаясь расписания)
4. Через 10-20 секунд обнови страницу — должен появиться запуск с зелёной галочкой (успех)
5. Если хочешь убедиться, что Telegram-алерты реально приходят — временно понизь любой порог в
   `monitor.py` почти до нуля (например, `FUNDING_RATE_THRESHOLD = 0.0000001`), закоммить,
   запусти workflow вручную ещё раз, дождись сообщения в Telegram, а потом верни порог обратно

## Как этим пользоваться дальше

- Ничего делать не нужно — раз в 15 минут GitHub Actions сам всё запускает
- Когда придёт алерт в Telegram — это сигнал зайти и вручную посмотреть контекст
  (что в новостях, что на TradingView, нет ли объявлений от проекта) — сама система решений не принимает
- Чтобы поменять пороги — просто отредактируй числа в начале `monitor.py` (переменные с `_THRESHOLD`)
  прямо в веб-интерфейсе GitHub (кнопка "карандаш" на файле) и сохрани — следующий запуск подхватит изменения
- Чтобы посмотреть историю срабатываний — вкладка Actions показывает лог каждого запуска (там же видно
  фактические цифры funding/OI/цены, даже если алерт не сработал)
- Если захочешь остановить — Settings → Actions → Disable actions, или просто удали workflow-файл
